# 古法手搓工艺与现代创新编程的结合的导航网站项目 ( 习作 )

基于纯原生 HTML/CSS/JavaScript 构建的高性能、极简风格静态导航网站。致力于用科学极简的纯粹代码，还原极致的 UI 质感与交互体验。

## ✨ 核心特性 (Features)

- **极致流畅 (Butter Smooth)**：内嵌 Lenis 平滑阻尼引擎，提供 60fps 原生级物理滚动体验。
- **光学级毛玻璃 (Progressive Blur)**：使用 CSS `mask-image` 实现硬件加速的渐进式毛玻璃遮罩，完美还原顶级视觉质感，告别渲染黑块。
- **DOM 降维隔离术 (Layer Decoupling)**：彻底解决 WebKit 引擎下 `mask-image` 与 `background-clip: text` 的底层死锁 Bug。采用父级 Alpha 通道消散与子级纹理裁切物理隔离，实现最纯净的文本镂空渐隐。
- **无限波浪履带引擎 (Staggered Wave Marquee)**：
  - **视觉障眼法**：基于 `50% Loop` 的双重/多重分身节点拼接，配合强硬的 `flex-shrink: 0` 刚体护盾，实现 4K 宽屏下严丝合缝的无限滚动。
  - **矩阵波浪**：通过 JS 在微秒级将文本碾碎解构为独立 DOM，注入 CSS 变量 `--i` 生成精准的延迟序列（Staggered Delay），触发大厂级的 hover 波浪翻转动效。
- **三明治渲染架构 (Sandwich Architecture)**：针对复杂悬浮组件（如 Back-To-Top 按钮），采用“底盘毛玻璃刚体 + 中层 GPU 匀速旋转履带 + 顶层绝对静止轴心”的 3 层独立渲染结界，杜绝 SVG 内部 `<foreignObject>` 引起的 Safari 渲染崩溃与边缘裁切。
- **全局物理垫片联动**：通过 CSS `:root` 变量引擎（如 `--main-spacer-top`），实现页面留白间距与毛玻璃透镜高度的纳米级自动咬合，告别断层。
- **智能滚动吸附与追踪**：侧边栏选中态使用 `transform` 动态追踪坐标（物理滑块），并实现双向 Scroll-spy 联动，杜绝重排 (Reflow) 性能损耗。
- **事件委托架构**：导航点击事件采用 O(1) 级绑定，支持无限量分类节点扩展，内存占用极低。
- **零构建工具 (Zero Build)**：标准静态目录结构，开箱即用，秒级部署。

## 🗺️ 架构演进路线图 (Architecture Roadmap)

为了推动界面结构与内容数据的彻底解耦，提升站点的可维护性，本项目规划了三个阶段的演进路线：

### 🟢 Lv.1: 静态纯享版 (当前阶段)
- **状态**：内容数据（分类、链接、描述）直接硬编码在 HTML 结构中。
- **核心目标**：探索并夯实核心视觉资产与动效规范。跑通物理滚动引擎、CSS 变量体系、响应式布局及自定义分类锚点的双向联动。

### 🟡 Lv.2: 纯前端伪 CMS 架构 (核心演进计划)
- **目标**：探索数据分离，在不依赖任何后端服务器的情况下，实现可视化的导航配置管理。
- **数据结构**：制定标准的 JSON/JS 对象（如 `window.SITE_CONFIG`），囊括全局设置、分类目录与卡片元数据。
- **核心工作流 (Workflow)**：
  1. **双重读取机制**：页面初始化时，优先读取浏览器 `LocalStorage` 中的用户配置；若为空，则降级读取本地的 `config.js` 默认文件。
  2. **可视化配置弹窗**：通过点击侧边栏底部的“设置”图标，唤起带有毛玻璃质感的配置面板（Modal）。在此面板中以表单形式增删分类、修改链接信息。
  3. **无刷新热重载**：在弹窗内点击“保存”后，数据即刻写入 `LocalStorage` 并触发前端原生 DOM 的重新渲染，实现所见即所得。
  4. **一键导出与固化**：在配置面板提供「导出 JSON」功能。利用浏览器原生 `Blob` API，将内存中的最新配置打包下载为 `config.js`。开发者使用该文件替换本地旧文件，通过 Git 推送即可完成线上站点的永久更新。
- **AI 协同优势 (Vibe Coding)**：这种高度解耦的数据结构对 Cursor/Trae 等 AI 工具极度友好。AI 无需理解庞大的 UI 逻辑，仅需基于 JSON 数据骨架，即可快速推演出表单渲染与 LocalStorage 存取逻辑。

### 🔴 Lv.3: 无头 CMS 动态驱动 (Future)
- **目标**：拥抱全动态的内容管理生态。
- **实现方案**：将数据源接入 Headless CMS（如 Strapi、Notion API 或 Serverless 数据库）。只需将 Lv.2 中的 `LocalStorage` 存取替换为真实的 API 请求，前端 UI 代码无需任何大幅改动即可平滑过渡。

## 🚀 部署指南 (Deployment)

本项目目前为纯静态架构，推荐使用 **GitHub Pages**、**Vercel** 或 **Netlify** 进行托管。配置更新后，只需将导出的最新 `config.js` 推送至代码仓库，CI/CD 即可实现秒级自动部署。
